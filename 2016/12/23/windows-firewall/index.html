windows防火墙设置

发表于 Dec 23 2016 | 分类于 计忆流年 |

最近几天电脑好像中了病毒，然后今天偶然发现windows防火墙搞得乱七八糟，已经不能开启服务了，然后花了两个小时时间开始各种找问题和解决。

我的问题是：

  1. 不能打开控制面板，一打开就出现windows资源管理器停止工作。
  2. 发现打印机共享不能使用，我可以ping通别人的电脑，别人不可以ping我的。应该是windows防火墙有问题.
  3. 想要写github博文的时候，发现hexo g不能使用，npm和hexo命令都已经失效，可能是太久没有使用的原因
  4. U盘中了快捷方式的病毒，打开U盘发现文件夹很多都变成了快捷方式。

    问题一的解决

    原因是有个脚本js文件坏了，开始-所有程序-启动-把脚本文件删掉即可。
    然后把对应的脚本目录里的文件也删掉，重启电脑。

问题二的解决

防火墙一直不能启动服务，启动就会报错，windows不能在本地计算机启动windows firewall，错误代码13.我怀疑是昨天乱下了农行插件的原因，把我的防火墙给强制关闭了。

在google上找解决方法，直接把防火墙策略全部移除：Cannot start the Window Firewall error code 13

I’ve got it working on Windows 10, finally. See screenshot.
a. Run>regedit, then go here

b. Delete all folders INSIDE FirewallPolicy (Do NOT delete the FirewallPolicy folder)

c. Open an admin CMD, and type: net start MpsSvc The service should now start.

问题三的解决

在Git bash敲下命令hexo g，居然出现上述错误信息，然后发现输入任何有关hexo的命令都是同样的错误，我又试着敲了npm这个命令，发现也是出错的，但是在系统命令行中输入node -v和npm -v,都正常显示了版本信息，于是我想估计是环境变量设置有问题，我在用户的环境变量PATH中添加了两个路径，分别是C:\Users\yohy\node_modules\hexo\bin和C:\Program Files\nodejs，再加上原来就添加的路径C:\Users\yohy\AppData\Roaming\npm，发现就解决这个问题了，可以正常修改博文，然后发布网站。

解决方法参考了：hexo使用问题总结

问题四的解决

电脑插入U盘以后发现文件夹都变成了快捷方式。参考了网上的解决方法

电脑插入U盘后里面的文件变成快捷方式

然而今天打开移动硬盘发现移动硬盘也中枪了，我心塞的心情无以复加啊，后来发现其实解决很简单，只需要把隐藏的文件显示出来

把快捷方式和vbs以及autorun.inf文件删掉，就可以了。

之后会发现很多正常文件夹也变成了受保护的操作系统文件，解决的方法是：

  1. 开始->运行->cmd，打开了命令提示符窗口，
  2. 然后输入：attrib -a -h -s -r 文件名，这个命令的意思就是说把在D盘下的123.txt这个文件的属性去掉存档、隐藏、系统、只读，其中a是指存档属性，h是指隐藏属性，s指系统属性，r指只读属性，+号指添加某一个属性，-号指去掉某个属性。
  3. 如果是系统文件变成了正常文件，则可以使用attrib -a -h -s -r 文件名。

噗，实验室是搞安全的，结果全实验室的人电脑都是裸奔哈哈哈，某天学妹插上她的U盘处理文件，结果把我的所有U盘移动硬盘全搞中枪了哈哈哈，终于get一项技能啊！本来还可以要格式化的，后来发现移动硬盘那么多剧不舍得删啊！
